## Фаза 1: MVP (done)

- setup.sh: fork + placeholders + launchd
- template-sync.sh: авто-синхронизация source → template
- validate-template.sh: 5 проверок генеративности
- CI (validate-template.yml) + GH Actions (scheduler)
- update.sh: пользователь тянет обновления

## Фаза 2: Уровни экзокортекса

> Пользователь получает L0 из коробки. L1 и L2 добавляются по желанию, когда появляется потребность.

| Уровень | Что включает | Когда добавлять | Как добавить |
|---------|-------------|-----------------|--------------|
| **L0** (текущий шаблон) | Claude Code + Стратег + governance hub (DS-strategy) | Сразу (fork + setup.sh) | Уже в шаблоне |
| **L1** | + Экстрактор (Knowledge Extraction) | Когда пользователь создаёт первый Pack | Пользователь сам клонирует DS-extractor-agent + настраивает промпты |
| **L2** | + Синхронизатор (cross-repo sync, code-scan) | Когда ≥3 downstream-репо | Пользователь сам клонирует DS-synchronizer + настраивает scheduler |

### Почему не всё сразу

- **Экстрактор** требует Pack (source-of-truth) — без него некуда извлекать знания
- **Синхронизатор** требует ≥3 репо — иначе нечего синхронизировать
- Принцип: **минимальная сложность на старте**, градиентное усложнение по мере роста
